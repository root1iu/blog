数组中重复的数据

给定一个整数数组 a，其中1 ≤ a[i] ≤ n （n为数组长度）, 其中有些元素出现两次而其他元素出现一次。找到所有出现两次的元素。(Leetcode-442)

Day Two

Hash

由于题目中有一个条件是1 ≤ a[i] ≤ n （n为数组长度），也就是数组的值与其索引的关系是

$i + 1 = a[i]$

那么可以使用一个布尔类型的Hash数组来判断是否重复，将Hash的值作为数组索引，Hash函数为$H\left( a\left[ i\right] \right) =a\left[ i\right] +1$ ，如果已经在数组的元素值对应的索引，在Hash数组中已经有记录，那么这个数字就是重复的数字。这种做法需要扫描一遍，并且需要开辟一个新的长度为n的布尔数组，所以时间复杂度和空间复杂度都为$O(n)$

Day Three

替换

替换的思路是根据索引和元素值应该是一一映射的关系。如果目前的索引和元素值不对应，那么就需要将目前的元素值换到目前元素值应该在的位置。具体的操作思路和优化思路如下：

  1. 每次遇到不在正确位置上的数，直接交换然后到下一个索引。这种操作思路没有考虑一点，就是如果是向前交换（往索引增加的方向），那么交换过来的元素是没有被扫描过的，这样会漏扫；

  2. 基于1的优化是每次交换后索引不动，将换过来的元素视为未元素值。出现什么问题：搜索到后面，可能会交换前面已经搜索过的数据，而搜索过的数据代表要么是正确的位置，要么是重复，既然这里是交换那么就意味着即将要交换过来的数据是已经判断过的重复数据，这时不能加入到结果中。也就是说往前交换的话，那么再次扫描的起点不应该是本节点，而应该下一个节点。
