数字的整数次方——Leetcode 50

实现 pow(x, n) ，即计算 x 的 n 次幂函数。

说明:

  • -100.0 < x < 100.0
  • n 是 32 位有符号整数，其数值范围是 [−231, 231 − 1] 。
思路1

直接循环乘.这种思路被n的范围给排除了..n太大的话时间不够.

思路2

利用n的二进制.具体思想是将数字想象成二进制,我们只需要将二进制中的’1’的数位对应的次方相乘即可.例如

具体做法是使用一个变量存储乘方的值,遇到数位为1则将该乘方与答案相乘,否则跳过.

注意:需要关注的是负数,负数则取其反码即可.(注意使用无符号变量,否则在右移时会出现符号位延伸的情况.),这里当然可以使用变量(初始化为1)来左移检测数位,但这样的话每个数据都需要左移32次.

特殊情况
  • n为0
  • n为负数
  • x为0,同时n为负数

超级次方——Leetcode 372

你的任务是计算 $a^b$ 对 1337 取模，a 是一个正整数，b 是一个非常大的正整数且会以数组形式给出。

思路

类似于数字的整数次方,这一题也类似,不过是从二进制转到10进制.比如

  • 二进制: $a=2, b=13$,其中$b=1101b$,所以$a^b = (a^{2^3})^1(a^{2^2})^1(a^{2^1})^0*(a^{2^0})^1$
  • 十进制: $a = 2, b = 13$,$a^b = (a^{10^1})^1*(a^{10^0})^3$

这样,就可以通过将指数按10进制数位拆分后进行计算.刚好题目已经将指数按10进制拆分好了,需要注意的计算过程中模运算的性质:$a^b \% m = (a \%m) ^ b$,利用该性质,需要对计算过程中的底数进行%运算避免溢出.
