UNPv1-Ch11-Name_And_Address_Conversions

发表于 2019-11-23 | 分类于 UNPv1

Domain Name System(DNS)

域名系统(DNS)主要是用来映射主机名和IP地址，主机名可以是简单域名，也可以是FQDN(fully qualified domain name)，前者是主机名的简称(如 solaris)，后者是主机名的全称(如 solaris.unpbook.com)。

资源记录(resource records, RRs)

A

A记录将一个主机映射成一个IPv4地址(32 bits)。

AAAA

AAAA记录将一个主机名映射成一个IPv6地址(128 bits)。

PTR

PTR(pointer records)记录将一个IP地址映射成主机名。对于IPv4的地址来说，4字节的地址被转换成4个ASCII码然后翻转(1.2.3.4 -> 4.3.2.1)，后面跟上.in-addr.arpa，如111.111.111.111.in-addr.arpa；对于IPv6的地址来说，16字节的地址被转换成16个ASCII码，后面添上.ip6.arpa。

MX

MX记录指定一个主机作为另一个主机的邮件交换器(mail exchanger)。

CNAME

CNAME就是规范名字(canonical name)，通常是为某个服务指派一个规范主机名，用户只需要知道服务而不需要知道主机名。

解析器和名字服务器

在一些系统上，解析器代码包含在系统库中，使用其功能的应用需要将代码链接到自己的应用上。而在其他系统，解析器是一个守护进程，系统库代码可以通过RPC(remote procedure call)来调用其服务。

解析器通常会使用UDP来发送DNS询问。如果其中的数据太大了，那么解析器会自动地转用TCP。

gethostbyname 函数

注意：

  1. 这个函数返回的内容是静态分配的，所以要注意可重入问题。
  2. 另外这个函数只支持IPv4地址。这个函数最好不要使用，因为POSIX标准建议不要在新代码中使用它，而且有更好的函数getaddrinfo。
  3. 这个函数的错误码是通过h_errno来返回的，这个错误码可以使用hstrerror来获得错误描述。

gethostbyaddr函数

addr是一个指针，指向一个in_addr结构体，len是其长度。family只能是AF_INET。

这个函数和gethostbyname功能一样，只不过这个函数是通过IP地址来搜索host。

getaddrinfo 函数族

getaddrinfo

getaddrinfo函数简单来说就是gethostbyname + getservbyname + IPv4&6的功能。

hostname可以是主机名也可以是IP地址，service可以是服务名也可以是端口号。

hints是一个提示结构体，可以为空，此时假设ai_family的值为AF_UNSPEC，其他的可设置提示为0；也可以设置不同的调用提示，比如可以通过设置ai_socktype来设置使用流传输还是数据报传输，等等。具体的可设置提示如下:

  • ai_flags是一些flags提示：
    • AI_PASSIVE，调用者将使用这个socket来创建一个被动开启socket(服务端)
    • AI_CANONNAME，告诉函数返回主机的规范名
    • AI_NUMERICHOST，告诉函数hostname是一个IP地址
    • AI_NUMERICSERV，告诉函数service是一个端口号
    • AI_V4MAPPED，如果设置了这个flag且ai_family为AF_INET6，那么在没有AAAA记录时返回与A记录对应的IPv4-mapped IPv6地址
    • AI_ALL，如果和AI_V4MAPPED一起被设置，那么在返回AAAA记录时会附加返回与A记录对应的IPv4-mapped IPv6地址
    • AI_ADDRCONFIG，只返回与设置的IP版本一致的IP地址
  • ai_family指明了IP的版本
  • ai_socktype指明了socket的类型，比如SOCK_STREAM
  • ai_protocol指明了协议类型

参数res是value-return参数，是一条链，其中ai_next是指向下一个节点的指针。

注意：大部分getaddrinfo的实现只有在ai_socktype设置了之后才会接受service为字符串形式的端口号，若没有设置ai_socktype且service为字符串形式的端口号，那么会报错(返回错误码-1)。[注意重现时参数hints不能为空，需要分配内存然后设置ai_socktype为0]

客户端

TCP的客户端就是遍历getaddrinfo返回的地址结构，一个一个尝试连接。UDP的客户端可以通过判断连接失败(连接的UDP socket)或超时(未连接的UDP socket)来判断尝试是否成功。

服务端

服务端通常会设置service而忽略hostname，且会在hints设置AI_PASSIVE。这样socket会创建一个通配地址socket。之后TCP服务器就可以进行socket -> bind -> listen，UDP服务器就可以进行socket -> bind -> recvfrom。

gai_strerror函数

getaddrinfo会将错误码返回，此时可以使用gai_strerror来将错误码转换成字符串。

freeaddrinfo函数

传入getaddrinfo函数的指针res的成员指针指向的内存块是getaddrinfo动态分配的，所以我们需要用freeaddrinfo来释放其中分配的内存。

getaddrinfo函数: IPv6

IPv6要注意一点是当ai_family设置为AF_INET时，getaddrinfo无法接受IPv6型的地址串，相同的，当ai_family设置为AF_INET6时，getaddrinfo无法接受IPv4型的地址串。但ai_family设置为AF_UNPSEC时，两种都可以被getaddrinfo接受。

getnameinfo函数

这个函数是getaddrinfo的互补，就是给一个地址结构，返回主机名和服务名。

flags的6种不同值的含义:

  • NI_DGRAM，当调用者想要找地址和端口对应的UDP服务时需要这个标识
  • NI_NAMEREQD，如果不能将地址解析为主机名，则返回错误
  • NI_NOFQDN，返回主机名而不是FQDN
  • NI_NUMERICHOST，返回host的IP地址而不是主机名
  • NI_NUMERICSCOPE，返回数字形式的scope identifier[疑问: 什么是scope identifier]
  • NI_NUMERICSERV，返回服务的端口号而不是服务名
