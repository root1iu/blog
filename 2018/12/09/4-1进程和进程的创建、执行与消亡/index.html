要素3 : task_struct

task_struct 内容

即PCB，进程在OS中的身份证。task_struct的定义如下：

volatile long state

int sigpending

表示进程收到了信号但尚未处理。【待补充】

int counter

与调度有关。 【待补充】

volatile long need_resched;

与调度有关，表示CPU从系统空间返回用户空间前夕需要进行一次调度。

mm_segment_t addr_limit

虚存地址空间的上限。对进城而言是用户空间的上限，即0xbfff ffff；对内核而言是系统空间的上限，即0xffff ffff。

struct exec_domain *exec_domain;

应用程序有版本间的差异，从而形成了不同的执行域。exec_domain

struct linux_binfmt *binfmt

应用程序的文件格式。

kernel_cap_t cap_effective, cap_inheritable, cap_permitted

操作权限。内核有一个函数capable()，可以用来检验进程是否具有某种权限。

struct user_struct *user

user_struct代表进程所属的用户。

TSS

Intel提供了TSS作为进程调度的硬件支持，但Linux并没有使用，因为使用TSS来调度进程开销会很大，使用 jmp 或者 call 指令进行任务切换需要300多个CPU时钟周期（《Linux内核情景分析》的数据）。所以Linux内核中，每个CPU只用一个TSS来满足硬件要求，但不使用任务门。

对Linux内核来说，TSS有用的只有0级栈指针。

线程和进程

如果缺少第4要素，即独立的存储空间，则称为线程。如果完全没有用户空间，则成为内核线程； 如果共享用户空间则称为用户线程。这里的线程是对操作系统而言的，和平常所用的在进程中实现的线程不一样，这里我的理解是平常所说的线程是依附于进程，使用进程的内存空间。（未经考证）

进程三部曲：创建、执行和消亡

进程如何创建

从父进程复制一个子进程出来，作为一个新进程。第一个父进程就是内核的init进程，这是所有进程的始祖。这里，复制有两种方式，Linux提供了两种系统调用，分别为fork和clone。还有一种vfork，不过vfork出来的是线程不是进程。

fork

fork是全部复制。父进程的所有资源子进程都有一份副本。

clone

clone是部分复制，拥有父进程的资源的部分副本，其他通过复制指针共享。

vfork

除了task_struct结构和系统空间堆栈是数据结构复制外，其他资源都通过复制指针共享。

执行

子进程的执行路径

  • 离开父进程——execve()
  • 走父进程接下来的路（fork出来的子进程不执行execve()）

父进程的执行路径

进程不受阻 —— 异步

  • 继续走自己的路
  • 退出——exit()

进程受阻 —— 同步

  • 停下来等待子进程返回
    • wait4() —— 等待某个特定的子进程返回
    • wait3() —— 等待任何一个子进程返回
